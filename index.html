Self-Assessment

Throughout my time in the Computer Science program at SNHU, I have grown significantly as a developer, a problem solver, and a collaborator. The diverse coursework, ranging from foundational programming in IT 140 and CS 210 to more advanced topics such as CS 465 (Full Stack Development I) and CS 499 (Capstone), has provided me with a broad and in-depth understanding of the field. Developing my ePortfolio has been a valuable opportunity to reflect on that learning, consolidate it into tangible demonstrations of my capabilities, and position myself effectively for future employment opportunities in the tech industry.
Completing the capstone and curating the enhancements into a single survival game project allowed me to unify the many threads of my education. The project draws on areas such as software engineering, data structures, database design, and game AI. It brings them together into a single artifact that evolves throughout the capstone. This artifact doesn’t just demonstrate isolated technical skills. Instead, it showcases my ability to take a system from concept to execution, apply enhancements iteratively, and communicate technical decisions with confidence.
From a technical standpoint, I have applied principles of data structures and algorithms, as taught in courses such as CS 300 and MAT 230, and demonstrated them in the game’s entity system, AI behaviors, and procedural terrain generation. My understanding of software engineering and databases matured significantly through DAD 220, CS 340, and CS 465, where I developed scalable backend components and practiced clean architecture principles. During the capstone, I used MongoDB to implement persistent world state and player data, adding a layer of durability and complexity to the system. Though security enforcement isn’t yet in place, I’ve considered future measures such as input validation, access control, and anti-cheat protections, which are especially relevant for commercial applications.

Artifacts

This ePortfolio showcases a single, multifaceted artifact, a procedurally generated 2D survival game, enhanced across three core domains of computer science: software engineering and design, algorithms and data structures, and database systems. Originally developed as a foundational project, it was significantly expanded in CS 499 to include complex gameplay systems, intelligent entity behaviors, and persistent world state management. By utilizing a single artifact across all enhancements, I demonstrated the deep integration of multiple competencies in a cohesive and evolving software project.

Original Artifact

The original artifact was a basic 2D visual simulation built in C++ as part of my work in CS 330: Computational Graphics and Visualization. It showcased a scene management system and rudimentary object rendering. The core focus was on understanding camera systems, class hierarchies, and interactive displays. However, it lacked scalability and interactivity.

Enhanced Artifact

The enhanced version evolved into a procedurally generated 2D survival game implemented in Java using the LibGDX framework. It now includes an ecosystem of AI-driven entities, a player controller, a dynamic lighting system, and a MongoDB-backed persistent world and player data. This transformation demonstrates applied knowledge in object-oriented design, game loop logic, database connectivity, and behavior-based AI, far surpassing the original scope and complexity.
